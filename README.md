# Goodbirds Maps

Goodbirds is a static web mapping project that publishes daily bird-sighting maps from the eBird API. The site is built with Python, GitHub Actions, GitHub Pages, Folium, Leaflet, and Playwright-based screenshot posting for Bluesky.

Live site: <https://goodbirds.org>

## What the site includes

### City notable-birds maps

City maps show recent eBird notable sightings within a configured radius around selected places. Each city has a stable public URL and a generated map under `docs/maps/`.

Examples:

- <https://goodbirds.org/cambridge/>
- <https://goodbirds.org/estero/>
- <https://goodbirds.org/maps/cambridge/index.html>

City maps are built from the matrix in `.github/workflows/build.yml`. Each matrix entry defines the city slug, display title, center point, radius rings, default search radius, output directory, and stable page URL.

### Coverage map

The coverage map shows all Goodbirds map locations on one Leaflet map.

URL:

- <https://goodbirds.org/cities_map.html>

The coverage map and `docs/cities.json` are generated by `scripts/make_cities_json.py`, which reads the city matrix from `.github/workflows/build.yml`.

### US + Canada Mega-Rarities

The mega-rarities map shows current ABA Code 4 and Code 5 bird species rarities across the ABA Area.

URL:

- <https://goodbirds.org/mega/>

This map is built by `scripts/build_mega_map.py` and the `.github/workflows/build-mega.yml` workflow.

### Target Species Maps

Target Species Maps are focused maps for selected species in selected places. They are separate from the regular city notable-birds maps.

Recommended URL pattern:

- <https://goodbirds.org/targets/>
- <https://goodbirds.org/targets/estero/>

Recommended file structure:

```text
docs/
  targets/
    index.html
    targets.json
    estero/
      index.html
      data/
        observations.json
```

The first target map is intended for the Estero, Florida area near Koreshan State Park and tracks selected species such as Burrowing Owl, Limpkin, Roseate Spoonbill, and Painted Bunting.

## Repository structure

```text
docs/
  index.html                    Site landing page generated by the build workflow
  CNAME                         Custom domain configuration for goodbirds.org
  cities.json                   Generated list of city map metadata
  cities_map.html               Generated coverage map
  goodbirds_logo_text.png       Site logo used by map pages

  maps/
    cambridge/
      index.html                Current generated city map
    estero/
      index.html                Current generated city map
    ...

  mega/
    index.html                  Generated ABA Code 4 and 5 mega-rarities map
    summary.json                Mega map build summary
    aba4.json                   ABA Code 4 species list
    aba5.json                   ABA Code 5 species list

  targets/
    index.html                  Target Species Maps index page
    targets.json                Target map metadata
    estero/
      index.html                Estero target-species map page
      data/
        observations.json       Generated target-species observation data

scripts/
  build_map.py                  Builds regular city notable-birds maps
  build_mega_map.py             Builds the US + Canada mega-rarities map
  make_cities_json.py           Generates cities.json and cities_map.html
  fetch_target_species.py       Fetches selected species for target maps
  capture_and_post_bsky.py      Captures map screenshots and posts to Bluesky

.github/workflows/
  build.yml                     Builds city maps and the landing page
  build-mega.yml                Builds the mega-rarities map
  build-target-species.yml      Builds Target Species Maps data
  post-map-to-bluesky.yml       Posts rotating city maps to Bluesky
  post-mega-to-bluesky.yml      Posts the mega map to Bluesky
  post-cities-map-weekly.yml    Posts the coverage map to Bluesky
  post-target-map-to-bluesky.yml Posts Target Species Maps to Bluesky
```

## Local setup

Create a virtual environment and install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Current dependencies:

```text
folium
requests
ipywidgets
playwright
atproto
```

For Bluesky screenshot workflows, install the Playwright Chromium browser locally:

```bash
python -m playwright install chromium
```

## Required secrets

GitHub Actions expects these repository secrets:

```text
EBIRD_API_KEY
BSKY_HANDLE
BSKY_APP_PASSWORD
```

`EBIRD_API_KEY` is required for map builds. `BSKY_HANDLE` and `BSKY_APP_PASSWORD` are required only for Bluesky posting workflows.

## Building maps

### Build one city map locally

Example for Cambridge:

```bash
export EBIRD_API_KEY="your_key_here"
export OUTPUT_DIR="docs/maps/cambridge"
export MAP_MAIN_TITLE="Cambridge, MA & Vicinity"
export CENTER_LAT="42.378500"
export CENTER_LON="-71.115600"
export DEFAULT_RADIUS_KM="20"
export BACK_DAYS="2"
export MAX_RESULTS="200"
export ZOOM_START="11"
export RING_KMS="5,10,15,20"

python scripts/build_map.py
```

### Build the coverage map locally

```bash
export GOODBIRDS_WORKFLOW_YAML=".github/workflows/build.yml"
export GOODBIRDS_BASE_URL="https://goodbirds.org"
python scripts/make_cities_json.py
```

This writes:

```text
docs/cities.json
docs/cities_map.html
```

### Build the mega-rarities map locally

```bash
export EBIRD_API_KEY="your_key_here"
export MEGA_BACK_DAYS_RECENT="1"
export MEGA_NATIONAL_MAX="60"
export MEGA_PER_SPECIES_MAX="0"

python scripts/build_mega_map.py \
  --aba_csv scripts/data/ABA_Checklist.csv \
  --taxonomy_csv scripts/data/eBird_taxonomy_v2024.csv \
  --out_dir docs/mega \
  --codes 4,5
```

### Build a target-species map locally

Example for Estero:

```bash
export EBIRD_API_KEY="your_key_here"
export CENTER_LAT="26.4317"
export CENTER_LNG="-81.8187"
export DIST_KM="32"
export BACK_DAYS="3"
export OUTPUT_DIR="docs/targets/estero/data"
export TARGET_SPECIES_JSON='[
  {"code":"burowl","name":"Burrowing Owl","color":"#8B4513"},
  {"code":"rosspo1","name":"Roseate Spoonbill","color":"#FF69B4"},
  {"code":"limpki","name":"Limpkin","color":"#FFD700"},
  {"code":"paibun","name":"Painted Bunting","color":"#4169E1"}
]'

python scripts/fetch_target_species.py
```

This writes:

```text
docs/targets/estero/data/observations.json
```

## GitHub Actions workflows

### City maps and landing page

`.github/workflows/build.yml`

Runs on a schedule and by manual dispatch. It:

1. Builds each city map from the matrix.
2. Writes each city map to `docs/maps/{slug}/index.html`.
3. Writes redirect pages such as `docs/cambridge/index.html`.
4. Regenerates `docs/cities.json` and `docs/cities_map.html`.
5. Regenerates the site landing page at `docs/index.html`.
6. Commits generated outputs back to the repository.

### Mega-rarities map

`.github/workflows/build-mega.yml`

Builds `docs/mega/index.html` and related JSON summary files.

### Target Species Maps

`.github/workflows/build-target-species.yml`

Recommended workflow for building selected-species maps. Each matrix row should define:

```yaml
slug: estero
title: "Estero, FL Target Species"
center_lat: "26.4317"
center_lng: "-81.8187"
dist_km: "32"
back_days: "3"
output_dir: "docs/targets/estero/data"
species_json: >
  [
    {"code":"burowl","name":"Burrowing Owl","color":"#8B4513"},
    {"code":"rosspo1","name":"Roseate Spoonbill","color":"#FF69B4"},
    {"code":"limpki","name":"Limpkin","color":"#FFD700"},
    {"code":"paibun","name":"Painted Bunting","color":"#4169E1"}
  ]
```

## Bluesky posting

Goodbirds uses `scripts/capture_and_post_bsky.py` to capture a map screenshot with Playwright and post it to Bluesky using `atproto`.

Posting workflows include:

```text
.github/workflows/post-map-to-bluesky.yml
.github/workflows/post-mega-to-bluesky.yml
.github/workflows/post-cities-map-weekly.yml
.github/workflows/post-target-map-to-bluesky.yml
```

The posting script supports direct overrides:

```text
FORCE_MAP_URL
FORCE_POST_TEXT
FORCE_ALT_TEXT
EXTRA_HASHTAG
DRY_RUN
WAIT_MS
```

Use `DRY_RUN=1` before enabling a new posting workflow publicly.

## Adding a new city map

1. Add a new city object to the matrix in `.github/workflows/build.yml`.
2. Include:
   - `slug`
   - `title`
   - `center_lat`
   - `center_lon`
   - `ring_kms`
   - `default_radius_km`
   - `maps_subdir`
   - `latest_href`
3. Run the workflow manually.
4. Confirm that the new map appears in `docs/maps/{slug}/index.html`.
5. Confirm that the short URL works, for example `https://goodbirds.org/{slug}/`.
6. Add the city to Bluesky rotation if desired.

## Adding a new Target Species Map

1. Create the target page directory:

```bash
mkdir -p docs/targets/new-target/data
```

2. Add or copy an `index.html` page to:

```text
docs/targets/new-target/index.html
```

3. Make sure the page fetches data by relative path:

```javascript
fetch("./data/observations.json")
```

4. Add the target to `docs/targets/targets.json`:

```json
{
  "slug": "new-target",
  "title": "New Target Species Map",
  "subtitle": "Species One, Species Two, Species Three",
  "region": "Region Name",
  "centerName": "Center Place Name",
  "radiusLabel": "20 mi",
  "lookbackLabel": "past 3 days",
  "url": "new-target/"
}
```

5. Add a matrix item to `.github/workflows/build-target-species.yml`.
6. Run the workflow manually.
7. Test the page at:

```text
https://goodbirds.org/targets/new-target/
```

8. Add the page to the target Bluesky workflow if desired.

## Notes and cautions

- `docs/index.html` is generated by `.github/workflows/build.yml`. Edit the workflow source if you want persistent homepage changes.
- Keep city notable-birds maps and Target Species Maps separate. For example, `/estero/` should remain the regular Estero notable-birds map, while `/targets/estero/` should be the Estero target-species map.
- Use relative paths inside target pages whenever possible, especially for `./data/observations.json`.
- Do not commit API keys, Bluesky app passwords, or other secrets.
- Run posting workflows with `DRY_RUN=1` before enabling public posts.

## Data attribution

Bird observation data comes from eBird and its contributors. Goodbirds links back to eBird checklists wherever possible.
